## 깃
- 소스 코드의 버전 관리 및 협업을 가능하게 해주는 도구

## 로컬 저장소(Local Repository)
- 내 컴퓨터에 있는 저장소

## 원격 저장소(Remote Repository)
- GitHub 서버에 있는 저장소

## fork
- 깃허브에서 다른 레포지토리를 내 원격 레포지토리로 복사

## clone
- 원격 저장소 파일들을 내 로컬 저장소로 복사

## branch
- 하나의 레포지토리내에서 여러 작업을 독립적으로 진행하기 위한 분기들

## 과제 제출 방법
1. 깃 repo fork하기
2. 해당 repo를 로컬로 내려받기
3. 작업 후 커밋, 푸쉬하기
4. 해당 작업사항을 개인 브랜치에 pr 올리기
