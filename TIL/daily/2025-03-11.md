## 브랜치(branch)

<aside>
💡

독립적으로 어떤 작업을 진행하기 위한 개념

</aside>

- 코드를 분기하여 관리하는 개념
- 여러 작업을 동시에 진행 가능

> 쇼핑몰을 만들고 있는데 새로운 기능이 필요하다
>

ex) 지금 작업폴더에서 쇼핑몰 만드는 코드를 짜고 있다고 가정

→ 쿠폰기능을 추가하고 싶을때

위험하고 복잡할 것 같다면 원래 있던 소스코드를 직접 수정하는게 아니라 프로젝트 사본을 만들어서 거기다가 먼저 개발해보는 것

→ branch 생성하여 프로젝트 복사해서 개발

### 주 명령어

- git init: 브랜치 생성 및 초기화
- git add: 파일을 인덱스에 등록
    - git add <파일명> : 해당 파일만 등록
    - git add . : 작업한 모든 파일 추적 등록
- git commit -m: 변경사항 기록
- git push: 원격 저장소에 업로드

- git status: 반영 현황 (추적되지 않은 파일 확인 가능)
- git log: 작업 로그
- git clone [저장소 경로]: 원격 저장소 복제
- git pull: 원격 저장소에서 불러오기 (병합)

### 커밋 컨벤션

- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- style:  코드 포맷팅, 세미클론 누락 등
- refactor: 코드 리팩토링
- test: 테스트 코드 추가 또는 수정
- !BREAKING CHANGE: 커다란 API 변경의 경우

- design: CSS 등 사용자 UI 디자인 변경
- comment: 필요한 주석 추가 및 변경
- rename: 파일 혹은 폴더명을 수정하거나 옮기는 작업
- remove: 파일을 사젝하는 작업
- !HOTFIX: 급하게 치명적인 버그를 고치는 경우
